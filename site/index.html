⬇ OpenRPC spec

Published profiles are the API shape only — method names, statuses, and a response signature: field structure with safe example values (numbers, enums like 5g) and format labels (<ipv4>, <mac>, <string>) standing in for anything identifying. MACs, serials, secrets, IPs, hostnames and SSIDs are scrubbed to labels — never published as values; raw response bodies are dropped entirely, and the registry validator rejects any submission that still contains one. Captured with glinet-profiler; a gli4py badge marks methods the gli4py client wraps.

Empirical observation, not an official vendor contract.
